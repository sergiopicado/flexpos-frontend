⚙️ Conexión Supabase
🏢 Datos del negocio (Emisor)
Estos datos aparecen en todas las facturas electrónicas como datos del Emisor.
🖥️ Este dispositivo (Sucursal y Terminal)
Identifica esta caja dentro de tu negocio. Si tienes varios POS, cada uno debe tener un número de terminal distinto para que los consecutivos no choquen en Hacienda.
Ejemplo: POS principal = Terminal 00001, FlexPOS = Terminal 00002.
🖼️ Logo del negocio (PDF)
El logo aparece en la esquina superior del PDF de facturas. Tamaño recomendado: 200×80px. Formatos: PNG, JPG.
🛡️ Seguridad — Row Level Security (Supabase)
Las políticas RLS ya están configuradas en las tablas creadas con los scripts SQL de FlexPOS. Para mayor seguridad, puedes crear políticas más restrictivas en Supabase → Authentication → Policies.
💡 Cajero: solo accede al POS e historial — no puede ver inventario, clientes ni reportes.
💡 Admin: acceso total desde cualquier dispositivo con sus credenciales.
💡 Para restringir el anon key a solo lectura del inventario, configura políticas RLS en Supabase con auth.uid().
🔗 Backend de firma (opcional)
⚠ El backend es necesario para firmar y enviar documentos a Hacienda. Sin él, descarga el XML y envíalo manualmente desde ATV.
💱 Tipo de Cambio BCCR (automático)
Regístrate gratis en gee.bccr.fi.cr para obtener tu token. El TC se actualiza automáticamente al iniciar sesión.
